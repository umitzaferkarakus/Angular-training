Key Notes (Lifecycle Hooks)

  • Do side effects in ngOnInit or the appropriate hook; avoid heavy work in the constructor.
  • For @Input changes, use ngOnChanges or consider reactive patterns.
  • Clean up subscriptions in ngOnDestroy to prevent memory leaks.
  • Avoid unnecessary ngDoCheck for performance.

Lifecycle Hooks Information

Constructor: Called when component is created
ngOnChanges: Called when input properties change
ngOnInit: Called after component initialization
ngDoCheck: Called during every change detection cycle
ngAfterContentInit: Called after content projection is initialized
ngAfterContentChecked: Called after content projection is checked
ngAfterViewInit: Called after component view is initialized
ngAfterViewChecked: Called after component view is checked
ngOnDestroy: Called before component is destroyed
