Key Notes (Dependency Injection)

  • Provider scopes: providedIn: 'root' (singleton), component/provider tree (hierarchical), and module/route‑scoped.
  • Use InjectionToken instead of interfaces; types are erased at runtime.
  • Keep services side‑effect free and easy to test when possible.

Pros

  • Loose coupling, easier testing and reuse.
  • Flexible configuration with hierarchical injectors.

Cons

  • Wrong scope may create unexpected multiple instances.
  • Cross‑dependencies increase maintenance cost.
