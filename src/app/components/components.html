Key Notes (Components)

  • Single Responsibility: each component should have a clear purpose.
  • Define clear contracts with Inputs/Outputs.
  • Choose Change Detection strategy (Default/OnPush) as needed.
  • With standalone components, declare dependencies locally.

Pros

  • Reuse, testability, and easy composition.

Cons

  • Poor naming and unclear contracts create confusion.
